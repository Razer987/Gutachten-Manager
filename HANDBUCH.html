🗂 Was ist der Gutachten-Manager?

Der Gutachten-Manager ist eine professionelle Webanwendung zur Verwaltung von unfallanalytischen und unfalltechnischen Gutachten. Er läuft vollständig lokal auf Ihrem Windows-PC — keine Cloud, keine Drittanbieter, Ihre Daten bleiben bei Ihnen.

🌐
Webanwendung

Läuft im Browser — kein separates Programm nötig

🔒
100% Lokal

Alle Daten bleiben auf Ihrem PC

🐳
Docker-basiert

Einfache Installation ohne technisches Know-how

💾
Persistente Daten

Daten bleiben erhalten, auch nach Neustart

Benötigte Software: Docker Desktop

ℹ️
Nur Docker Desktop wird benötigt

Der Gutachten-Manager bringt alles andere (Datenbank, Server, etc.) mit. Sie müssen nur Docker Desktop installieren.

Docker Desktop installieren

  1. Öffnen Sie https://www.docker.com/products/docker-desktop im Browser
  2. Klicken Sie auf "Download for Windows"
  3. Führen Sie die heruntergeladene .exe Datei aus
  4. Folgen Sie dem Installationsassistenten (alle Standardeinstellungen übernehmen)
  5. Nach der Installation: Windows neu starten
  6. Docker Desktop starten — warten bis das Wal-Symbol in der Taskleiste erscheint
✅
Docker ist bereit, wenn...

...das Wal-Symbol (🐳) in der Taskleiste unten rechts erscheint und "Docker Desktop is running" anzeigt.

📥 Installation

Schritt 1 — Projektordner herunterladen

Öffnen Sie die GitHub-Seite des Projekts und laden Sie den Projektordner als ZIP herunter:

  1. GitHub-Seite öffnen: https://github.com/Razer987/Gutachten-Manager
  2. Klicken Sie auf den grünen Button "Code"
  3. Wählen Sie "Download ZIP"
  4. ZIP-Datei speichern (z.B. auf dem Desktop)
  5. Rechtsklick auf die ZIP → "Alle extrahieren..."
  6. Zielordner wählen (z.B. C:\Gutachten-Manager\)
⚠️
Ordnerpfad ohne Sonderzeichen

Verwenden Sie einen einfachen Pfad ohne Leerzeichen oder Umlaute. Gut: C:\Gutachten-Manager — Schlecht: C:\Meine Dokumente\Ärzte Gutachten

Schritt 2 — Konfiguration einrichten

Im Projektordner befindet sich eine .env.example Datei. Erstellen Sie eine Kopie davon und benennen Sie sie .env:

# Im Explorer: .env.example kopieren und umbenennen in .env
# ODER: STARTEN.bat macht das automatisch beim ersten Start
💡
Automatische Konfiguration

Wenn Sie einfach STARTEN.bat doppelklicken, wird die Konfiguration automatisch erstellt. Sie müssen nichts manuell tun.

Schritt 3 — Fertig!

Das war es. Starten Sie jetzt die Anwendung wie unten beschrieben.

▶️ Starten & Stoppen

Im Projektordner befinden sich drei ausführbare .bat Dateien:

▶️
STARTEN.bat

Startet den Gutachten-Manager. Beim ersten Mal dauert es 5–15 Minuten (Download & Build).

⏹️
BEENDEN.bat

Stoppt alle laufenden Container. Ihre Daten bleiben erhalten.

📊
STATUS.bat

Zeigt den aktuellen Status und die letzten Log-Einträge an.

Erster Start (5–15 Minuten)

  1. Stellen Sie sicher, dass Docker Desktop läuft (Wal-Symbol in Taskleiste)
  2. Öffnen Sie den Projektordner im Explorer
  3. Doppelklicken Sie auf STARTEN.bat
  4. Falls Windows fragt "Möchten Sie diese App ausführen?" → Ja klicken
  5. Warten Sie, bis das Fenster meldet "Die Anwendung ist bereit!"
  6. Der Browser öffnet sich automatisch
⏱️
Erster Start vs. Folgestarts

Der erste Start dauert 5–15 Minuten (Basis-Images werden heruntergeladen, der Quellcode wird kompiliert). Alle weiteren Starts dauern nur 30–60 Sekunden.

Normaler Start

  1. Docker Desktop starten und warten
  2. STARTEN.bat doppelklicken
  3. Browser öffnet sich automatisch auf http://localhost

Stoppen

  1. BEENDEN.bat doppelklicken
  2. Warten bis das Fenster "erfolgreich gestoppt" meldet
  3. Docker Desktop kann danach ebenfalls beendet werden
🔒
Daten sind sicher

Wenn Sie die Anwendung stoppen, gehen keine Daten verloren. Alle eingetragenen Gutachten, Bilder und Dokumente bleiben erhalten.

💾 Daten & Backup

Wo werden meine Daten gespeichert?

Alle Daten werden in sogenannten Docker Volumes gespeichert. Das sind besondere Ordner, die Docker verwaltet und die auch dann erhalten bleiben, wenn Sie die Anwendung stoppen.

DatentypVolume-Name
Datenbank (alle Gutachten)gutachten_postgres_data
Hochgeladene Dateiengutachten_uploads
Automatische Backupsgutachten_backups
Log-Dateiengutachten_logs

Automatisches Backup

🔄
Tägliches Backup um 02:00 Uhr

Der Gutachten-Manager erstellt automatisch jede Nacht um 02:00 Uhr ein Backup der Datenbank. Backups werden 30 Tage lang aufbewahrt und dann automatisch gelöscht.

Manuelles Backup erstellen

Öffnen Sie ein Eingabeaufforderungsfenster (cmd) im Projektordner und führen Sie aus:

docker exec gutachten_api node -e "require('./dist/jobs/backup.job').runBackupNow()"

Daten beim Deinstallieren behalten

⚠️
Wichtig bei Deinstallation

Wenn Sie Docker Desktop deinstallieren oder die Volumes manuell löschen, gehen alle Daten verloren. Erstellen Sie vorher immer ein manuelles Backup!

🔧 Fehlerbehebung

Problem: "Docker ist nicht gestartet"

Fehlermeldung: Cannot connect to the Docker daemon

Lösung: Starten Sie Docker Desktop. Warten Sie, bis das Wal-Symbol in der Taskleiste erscheint. Dann STARTEN.bat erneut ausführen.

Problem: Browser öffnet sich, aber Seite lädt nicht

Seite zeigt "Diese Website ist nicht erreichbar"

Ursache: Die Container brauchen noch etwas Zeit zum Starten.
Lösung: 30–60 Sekunden warten, dann Browser-Seite neu laden (F5).

Problem: Port 80 ist bereits belegt

Fehlermeldung: bind: address already in use

Ursache: Ein anderes Programm (z.B. IIS, ein anderer Webserver) verwendet Port 80.
Lösung 1: Das andere Programm beenden.
Lösung 2: In der Datei infrastructure/docker-compose.yml die Zeile "80:80" zu "8080:80" ändern. Dann läuft die App unter http://localhost:8080.

Problem: Erster Start schlägt fehl

Build-Fehler beim ersten Start

Lösung:
1. STATUS.bat ausführen und Fehler lesen
2. BEENDEN.bat ausführen
3. STARTEN.bat erneut ausführen
4. Sicherstellen, dass eine Internetverbindung besteht (für ersten Start nötig)

Logs anzeigen

Für detaillierte Fehlerdiagnose: STATUS.bat doppelklicken. Es zeigt die letzten 20 Log-Einträge aller Container.

Kompletter Neustart (Letzter Ausweg)

⚠️
Achtung: Löscht alle Daten!

Nur als absoluten letzten Ausweg nutzen. Erstellen Sie vorher ein Backup!

# Eingabeaufforderung im Projektordner öffnen:
docker compose -f infrastructure/docker-compose.yml down -v
# Dann STARTEN.bat erneut ausführen

❓ Häufige Fragen (FAQ)

Muss Docker Desktop immer laufen, wenn ich die App nutze?
Ja. Docker Desktop muss gestartet sein, bevor Sie STARTEN.bat ausführen. Sie können Docker Desktop so konfigurieren, dass es automatisch mit Windows startet (Einstellungen → General → "Start Docker Desktop when you log in").
Kann ich die App von einem anderen PC im Netzwerk aufrufen?
Ja, aber nur im lokalen Netzwerk. Andere PCs können die Anwendung unter http://[IP-Adresse-Ihres-PCs] aufrufen. Die IP-Adresse finden Sie über Eingabeaufforderung → ipconfig.
Was passiert mit meinen Daten wenn ich den PC neu starte?
Alle Daten bleiben erhalten. Nach dem Neustart müssen Sie lediglich Docker Desktop starten und dann STARTEN.bat ausführen.
Wie aktualisiere ich auf eine neue Version?
1. BEENDEN.bat ausführen
2. Neue Version herunterladen und in denselben Ordner entpacken
3. STARTEN.bat ausführen — das System baut automatisch neu
Ihre Daten bleiben dabei erhalten.
Wie ändere ich das Datenbankpasswort?
1. BEENDEN.bat ausführen
2. Datei .env mit Editor öffnen
3. DATABASE_PASSWORD ändern
4. STARTEN.bat ausführen
Achtung: Bei bestehender Datenbank muss das Passwort auch in der Datenbank geändert werden. Bei Neuinstallation reicht die .env-Änderung.
Warum öffnet sich kein Browser automatisch?
Falls der Browser nicht automatisch öffnet, geben Sie manuell http://localhost in die Adressleiste Ihres Browsers ein.
